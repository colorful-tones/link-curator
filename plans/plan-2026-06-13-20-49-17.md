# Link Curator MVP Plan Update

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build a local-first Astro app for saving links, enriching them with AI metadata, searching them, and exporting entries as clean Markdown.

**Architecture:** Use Astro in server mode with a local SQLite database under `data/`. Keep AI behind a provider-agnostic service that accepts OpenAI-compatible base URLs so open models can be used. Treat Markdown export as a first-class output path, not an afterthought.

**Tech Stack:** Astro, TypeScript, pnpm, SQLite, Vitest, Cheerio, OpenAI-compatible chat completions, Markdown export utilities.

---

## Decisions

1. Local-first only for now.
   - No deployment work.
   - No auth.
   - No hosted database.

2. Markdown export matters.
   - Each entry should export cleanly to a single Markdown document.
   - Include frontmatter for title, URL, tags, dates, source, content type, and summary.
   - Keep the body readable in Obsidian or any Git-backed notes repo.

3. Tags serve two purposes.
   - Personal knowledge system: durable concepts Damon may reuse in Obsidian.
   - Public discovery: simple tags that make sense to other readers.
   - Store both when possible as `personalTags` and `publicTags`.

4. Open models should work.
   - Do not hardcode OpenAI.
   - Use OpenAI-compatible config names: `AI_BASE_URL`, `AI_API_KEY`, `AI_MODEL`.
   - Link saving should still work if AI is disabled or fails.

5. Keep the build lightweight.
   - Avoid queues, background workers, user systems, vector DBs, and framework sprawl.
   - Prefer boring modules and explicit tests.
